Activating The Built In D-pad

Brass Monkey includes a D-pad component that can be activated. The d-pad component saves the developer time when creating this common arrangement of directional buttons. A pre-made set of d-pad graphics is included and they can be replaced at run-time. The D-pad is the component on a game controller that you use to navigate your game-avatar/cursor, and is often coupled with 'action' buttons to jump or attack. The data stream that a host receives from the device is a Point with x/y values that is broadcast on changes to the user input. It combines characteristics of touch events and buttons to streamline a frequently coded user interface.

In this demo, we will build a control scheme from scratch, and replace the built in graphics with bitmaps created from simple movie-clips. The bitmaps could be from any source security domain allows.

//setting up a scheme manually for screen resolution 480 x 320

        StageScaler.LONG=480;

        StageScaler.SHORT=320;

        StageScaler.SetLandscape();

//creating a control scheme that fully fills the screen resolution.

        scheme=new AppScheme(480,320,"landscape",false,false,"linear");

//setup rectangles for componentes.

//We'll Create separate visual and hit rectangles for each one except the background

        var backgroundRect:Rectangle=new Rectangle(0,0,480,320);

        var dpadTouchRect:Rectangle=new Rectangle(12,20,252,282);

        var dpadVisibleRect:Rectangle=new Rectangle(44,75,180,180);

With the rectangles laid out, create the components.

        //create a background with pre-made movie clip.

        var bg:BMImage=new BMImage(StageScaler.ScaleFlashRect(backgroundRect,true)

                ,new Background());        

        bg.name="background";

        //The page variable helps with automating view stacks

        //using scheme-page-To-String(i) with dynamic control updates.

        //We want these to all be on the defualt top view, page 1.

        bg.page=1;

        //Add to the control scheme.                        

        scheme.addChild(bg);

Now create the d-pad component. Use the StageScaler to translate the pixel rectangle to the control scheme rectangle. The hitrect is optional to allow the activation area to be different than the visual area. The dead zone is a percentage of the center that is considered neutral. Setting radial to true will slice the input area like a pie. Setting it to false will calculate input using a tic-tac-tow 3 by 3 grid.

        //create a d-pad.

        var dpd:BMDPad= new BMDPad();

        //set scheme frame/page.

        dpd.page=1;

        //apply rects.

        dpd.hitRect=StageScaler.ScaleFlashRect(dpadTouchRect,false);

        dpd.rect=StageScaler.ScaleFlashRect(dpadVisibleRect,false);                        

        //alter the size of the dead zone.

        dpd.deadZone=30;

        //use radial or box mode.

        dpd.radial=true;

If the bitmap data is missing for any of these properties below, the client will use the on-board default graphics. The graphics provided for the inactive state and all directions must be the same size, and the active-state for each direction should include the entire d-pad rather than discrete slices.

        //draw movieclips and set bitmap data for the d-pad states.

        //inactive/up-state                        

        dpd.inactive = dpd.draw( new PadImage());

        //direction down, down-state.

        dpd.down = dpd.draw(new PadBtnDnDown());

        //direction left, down-state.

        dpd.left = dpd.draw(new PadBtnLtDown());

        //direction left-Down, down-state.

        dpd.leftDown = dpd.draw(new PadBtnDnLtDown());

        //direction left-Up, down-state.

        dpd.leftUp = dpd.draw(new PadBtnUpLtDown());

        //direction right, down-state.

        dpd.right = dpd.draw(new PadBtnRtDown());

        //direction right-Down, down-state.

        dpd.rightDown= dpd.draw( new PadBtnDnRtDown());

        //direction right-Up, down-state.

        dpd.rightUp = dpd.draw(new PadBtnUpRtDown());

        //direction up, down-state.

        dpd.up = dpd.draw(new PadBtnUpDown());                                        

        //add to scheme.

        scheme.addChild(dpd);

To receive d-pad updates from a target device, add a d-pad event listener.

        device.addEventListener(DeviceEvent.DPAD, onDPad);

The value of the d-pad device event will be a DpadUpdate object. As the client changes the d-pad input, it sends the current direction as a point with x/y values set to -1,0, or 1. The y value -1 represents up and the x value -1 represents left.

        protected function onDPad(event:DeviceEvent):void

        {

                var update:DPadUpdate=event.value as DPadUpdate;         

                //get current d-pad direction intent.

                direction.x=update.x;

                direction.y=update.y;                        

                //process the input.

                process();

        }

Determining the management of several directional button-states or calculating radial touches within the host is replaced by interpreting a point object.

        public function process():void

        {

                        

                var dir:String="";

                        

                if(direction.y<0 )

                {                                                                                        

                        dir="n";

                }

                else if(direction.y>0 )

                {                                                        

                        dir="s";

                }

                        

                if(direction.x>0 )

                {                                                                                                

                        dir+="e";

                }

                else if(direction.x<0)

                {                                

                        dir+="w";

                }

        }
