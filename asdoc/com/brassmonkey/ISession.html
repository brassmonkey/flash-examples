connectDevice()method
public function connectDevice(device:Device):void

Initiate a connection to a device that has requested one.

The device must first have prepared a port for the incoming host. If you do not add event listeners to the BMApplication for device connection request, the application will connect auto matically when the total client count is less than the current client maximum. If you do add the listener, you must use this method to connect to the device. A host does not have to respond to a connection request.

Parameters

device:Device

getSlotDisplay()method 
public function getSlotDisplay():SlotDisplay

Get the display object that represents the slot id of this session.

Imagine the player is looking at two identical game screens with a friend, and is ot sure which game he see's in the phone is the his screen or his buddy's screen. If a device asks a host to show it's slot id, it is needing a visual cue on the stage of what that slot is. The slot display object is a handy sprite you can pop up when the request comes. The request can also be ignored if your host is already in a game state with another player.

setKeyboardMode()method 
public function setKeyboardMode(device:Device, startString:String):void

Request keyboard input.

When setting a device into Keypad Mode for specific input such as 'username' etc..., add a listener to that specific device for the DeviceEvent.TEXT_ENTER event, and then retrieve the text content using device.textContent. After getting the submitted text content, remove the event listener from the device.

Parameters

device:Device
 
startString:String — The content the device sill start with.

setWaitForHost()method 
public function setWaitForHost(device:Device, hostDeviceId:String):void

Set the device in a state which will make it wait for and connect to, a known host-device id that is expected to be coming online shortly.

After leaving the second host, the device will attempt to re-connect to the original host.

Parameters
