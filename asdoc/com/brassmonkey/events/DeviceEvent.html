DEVICE_DISCONNECTEDConstant 
public static const DEVICE_DISCONNECTED:String = deviceDisconnected

A device has closed it's two-way pipe to the host and is considered to have left the application session.

If the device was sent by another host, it will return. If the second host interupted the chain by sending the device to a third host, the second host should send the client-device back to the first, if the first host is known. The main concept is that at the end of a game/client-session, the host should return the client to the main portal, and then close. This is an automatic process as long as your own host does not send the device to another. It is only when you over-write the primary host by doing your own transfer to the third host. When the third host closes, the devices return to the second host. It no longer has a sense of the primary host. The server registry provides a list of hosts if requested, so that the portal can be found and the client routed.
