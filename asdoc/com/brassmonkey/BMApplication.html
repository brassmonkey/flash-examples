Packagecom.brassmonkey
Classpublic class BMApplication
InheritanceBMApplication flash.events.EventDispatcher
Implements IBrassMonkey

Application developers can extend this class and override the handler functions, or add event listners to handle BMEvents.

By adding event listeners for device events such as TouchEvent or InvokeEvent, the listener will receive events from all connected device. The listener will also receive all key events from all devices. However, when setting a device into Keypad Mode for specific input such as 'username' etc..., add a listener to that specific device for the DeviceEvent.TEXT_ENTER event, and then retrieve the text content using device.textContent. After getting the submitted text content, remove the event listener from the device.

Pass the flash vars or loader info parameters to the constructor. When lauched from the portal, it will provide a recommeded device id to use. Not using the recommened value for device id will cause your application to not integrated with the portal correctly.

deviceDisconnected Event  
Event Object Type: com.brassmonkey.events.DeviceEvent

A device has closed it's two-way pipe to the host and is considered to have left the application session.

If the device was sent to this app by another host, it will return to the sending host if present. If the second host interupted the chain by sending the device to a third host, the second host should send the client-device back to the first, if the first host is known. The main concept is that at the end of a game/client-session, the host should return the client to the main portal, and then close. This is an automatic process as long as your own host does not send the device to another. It is only when you over-write the primary host by doing your own transfer to the third host. You must provide the flash var 'bmPortalId' that reflects your host device id, or the one given to your application to the third host .
